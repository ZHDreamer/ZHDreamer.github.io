27 Special Topics II (CS Theory)

Complexity

P vs NP

NP: “nondeterministic polynomial time”, types of problems when given a “solution” can be verified if the solution is correct quickly.

如果一个问题有一个解，且能在多项式时间内证明这是个正解，那么这就是个NP问题

P: “polynomial time”, types of problems that we can create solutions to quickly.

如果一个问题是P问题，那么它必须满足在多项式时间nc内验证一个算法问题的实例是否有解 ，其中n是输入长度，c是个常数

P is a subset of NP, if we can create a solution in polynomial time, then we can verify it in polynomial.

Reductions

We say problem A reduces to problem B if you can easily solve problem A using problem B.

This implies that problem B must be at least as hard to solve ass problem A. e.g. using a more complicate function to solve a relative easy problem.

NP Complete

There are a group of problems in NP which all NP problems reduce to, we call this group NP complete.

Implications

Because all problems in NP reduce to NP-complete problems if an efficient solution to an NP-complete problem is found all problem in NP subsequently will have an efficient solution.

If P=NP, every mathematic problem will be easily solved.

Computability

A Classic Proof (Turing, 1936)

Can we check to see if a program will always terminate?

Proof by Contradiction

Let’s say we have a function, halt, which takes in a function f and value x, and returns True if f will terminate when called on x.

Then we can write a function:

def foo(f):
	if halt(f, f):
        while True:
            print("Trapped")
    else:
        return "Treminated!"

What might happen when we call foo(foo)?

  1. It terminates - so halt(foo, foo) returns False, which means foo(foo) doesn’t terminate
  2. It’s trapped forever - so halt(foo, foo) returns True, which means foo(foo) terminates

So that is a paradox. Neither case makes sense, so halt must not exist!

The halting problem is undecidable, which means it is impossible for a computer to solve.

The function halt reduces to is also undecidable.

Termination Analysis

Given a program p and an input x, return True if you know it halts, False if you know it doesn’t terminate, or “I don’t know” otherwise.

Goal: Minimize the number of times you have to say “I don’t know”

Another Classic Proof (Turing 1949)

Imagine we have a function f which takes in the variables of the program and spits out a number. If that number keeps decreasing, eventually it must hit some “smallest” value. If it hits the smallest value, our program terminates.
