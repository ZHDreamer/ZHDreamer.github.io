23 Streams

Streams

Lazy evaluation

(define (ints first)
    (cons first
          (ints (+ first 1))
    )
)
scm> (ints 1)
maximum recursion depth exceeded

(define (ints first)
    (cons-stream first
                 (ints (+ first 1))
    )
)
scm> (ints 1)
(1 . #[promise (not forced)])

cdr can not force scheme to evaluate the rest of the stream, but cdr-strem can.

A stream is just a regular Scheme pair whose second element is a promise.

Promises

Promise: an object that delays evaluation of an expression

  • The delay special form creates promises
  • The force procedure evaluates the expression inside the promise
scm> (define x (delay (print 5)))
x
scm> x
#[promise (not forced)]
scm> (force (delay (print 5)))
5
scm> x
#[promise (forced)]

cons-stream and cdr-stream are syntactic sugar. Achieve the same effect with delay and force.

Recursively defined streams

Constant stream

A constant stream is an infinitely long stream with a number repeated.

(define (constant-stream i)
    (cons-stream i (constant-stream i))
)

Natural number stream

(define (nats start)
    (cons-stream start (nats (+ start 1)))
)

You can also define it this way:

(define (add-stream s1 s2)
    (cons-strsam (+ (car s1) (car s2))
                 (add-stream (cdr-stream s1) (cdr-stream s2))
                 )
)
(define ones (cons-stream 1 ones))
(define ints (cons-stream 1 (add-stream ones ints)))

Map-stream

(define (map-stream fn s)
    (if (null? s)
        nil
        (cons-stream (fn (car s))
                     (map-stream fn (cdr-)))
    )  
)
