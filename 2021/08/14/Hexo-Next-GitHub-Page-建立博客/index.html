Hexo+Next+GitHub Page 建立博客

虽然 Hexo 和 Next 都有完整的 doc 供你参考，但我觉得还是有必要写这样一篇文章帮助我以及可能会看这篇文章的你记录一下我搭建的过程，以及踩过的坑。

安装依赖

安装Git

Git下载链接

Git是开源的分布式版本控制系统，用于将网站上传到GitHub。

下载安装后右键资源管理器会出现Git Bash Here，Git Bash是Git自带的命令行工具，可以在里面打命令（CMD和PowerShell也是可以的）

配置Git环境

设置user.name和user.email配置信息

git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"

配置SSH

ssh-keygen -t rsa -C "你的GitHub注册邮箱"

之后一路回车就好，生成后可以到c:\users\yourname\.ssh中找到id_rsa.pub

复制里面的内容（不要用微软Publisher打开，用Editor打开），打开 GitHub Setting Keys 页面，新建new SSH Key

Title是标题，填入任何你想填的内容即可，Key中复制刚才id_rsa.pub的内容，点击Add SSH Key即可生成SSH key。

在命令行输入

ssh git@github.com

提示

PTY allocation request failed on channel 0
Hi ZHDreamer! You've successfully authenticated, but GitHub does not provide shell access.
Connection to github.com closed.

证明配置成功

GitHub要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否是合法用户，为了省去每次输入密码的步骤，采用了SSH，当你推送的时候，Git就会匹配你的私钥跟GitHub上面的公钥是否是配对的，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是正确合法的。

安装Node.js

Node.js下载地址

Hexo基于Node.js，安装后输入

node -v
npm -v

验证是否成功安装Node.js和npm

Hexo 的安装与使用

安装Hexo

使用npm命令安装Hexo

npm install -g hexo-cli

安装完成后就可以用Hexo命令建立博客了

hexo init <folder>
cd <folder>
npm install

此时输入

hexo s

即可在localhost:4000看到建立的博客了

推送到GitHub

官方文档

安装 hexo-deployer-git

npm install hexo-deployer-git --save

在根目录下有一个_config.yml文件，是Hexo的==站点配置文件==，更改deploy配置

deploy:
  type: git
  repo: https://github.com/ZHDreamer/ZHDreamer.github.io.git
  branch: master

Hexo的设置

官方文档

NexT安装与使用

官方网站 ，其他网站的信息或多或少有些过时，包括这个网站和GitHub上面的教程

下载NexT

git clone https://github.com/next-theme/hexo-theme-next themes/next

启用NexT

在站点配置文件中修改

theme: next

此时输入hexo clean清理缓存后再次启动本地服务器，即可看到主题变为NexT

NexT的设置

在theme\next下有一个_config.yml文件，是NexT的==主题配置文件==，在这个文件中修改NexT设置

更改NexT主题

NexT目前提供四种主题，在==主题配置文件==

# Schemes
scheme: Muse
#scheme: Mist
#scheme: Pisces
#scheme: Gemini

去掉主题配置文件中的注释即可切换主题

添加tags,categories等页面

在==主题配置文件==

# ---------------------------------------------------------------
# Menu Settings
# ---------------------------------------------------------------

# Usage: `Key: /link/ || icon`
# Key is the name of menu item. If the translation for this item is available, the translated text will be loaded, otherwise the Key name will be used. Key is case-senstive.
# Value before `||` delimiter is the target link, value after `||` delimiter is the name of Font Awesome icon.
# External url should start with http:// or https://
menu:
  home: / || fa fa-home
  #about: /about/ || fa fa-user
  #tags: /tags/ || fa fa-tags
  #categories: /categories/ || fa fa-th
  #archives: /archives/ || fa fa-archive
  #schedule: /schedule/ || fa fa-calendar
  #sitemap: /sitemap.xml || fa fa-sitemap
  #commonweal: /404/ || fa fa-heartbeat

# Enable / Disable menu icons / item badges.
menu_settings:
  icons: true
  badges: false

去掉注释即可解锁相应页面

hexo new page "tags"

在source\下会生成一个tags文件夹，里面有一个index.md，添加type: "tags"

---
title: tags
date: 2021-08-13 16:53:54
type: "tags"
---

其他类型的页面都可以这样操作
